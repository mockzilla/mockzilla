Your API, simulated in seconds!

Mockzilla provisions a dedicated simulation server from your OpenAPI specs on every push. No config files. No shared sandboxes. Just a real URL your team can hit immediately.

How it works

  1. Push or open a PR — your specs are packaged and uploaded automatically
  2. A dedicated simulation is provisioned in your nearest region
  3. Your simulation is live at api.mockzilla.org/{org}/{repo}/{ref}/

Explorer

  • Browse all routes from your OpenAPI spec in the sidebar
  • Fire requests directly from the UI and inspect responses
  • View request history with full headers, body, and timing
  • Replay any recorded request instantly — tweak parameters and compare responses
  • Override upstream URL, latency, caching, and replay behavior per-request
  • Set custom headers and save reusable request presets

Simulation behavior

  • Responses generated from your spec schemas — realistic types, formats, and constraints
  • Fake data helpers: {fake:uuid.v4}, {fake:internet.email}, and more
  • Configurable latency to simulate slow upstreams or network conditions
  • Mimic error responses and specific status codes on any route
  • Proxy to a real upstream — Mockzilla falls back to simulation automatically when it is unavailable
  • Combine multiple OpenAPI specs into a single unified API surface
